#### SQL의 분류

DL : 데이터 랭귀지 DDL은 정의관련 DML은 조작기능 DCL은 제어(권환과 관련된) 기능  

- DDL의 종류 : CREATE(생성), DROP(제거), ALTER(수정)
- DML의 종류 : SELECT(검색), INSERT(삽입), DELETE(삭제), UPDATE(갱신)
- DCL의 종류 : GRANT(권한부여), REVOKE(권한해제), COMMIT(완료), ROLLBACK(복귀)

##### DDL

- 데이터베이스의 객체를 대상으로 정의/변경/삭제에 사용되는 언어이다.
- 논리적 데이터 구조와 물리적 데이터 구조로 정의할 수 있다.
- 논리적 데이터 구조와 물리적 데이터 구조 간의 사상을 정의한다.
- 번역한 결과가 데이터 사전에 저장된다.
- CREATE : 스키마, 도메인, 인덱스 정의
- ALTER : 테이블 정의 변경 (열)
- DROP : 스키마, 도메인, 인덱스 삭제

###### DROP (CASCADE vs RESTRICT)

- DROP View : View_이름 [CASCADE | RESTRICT];
- CASCADE   : 삭제할 요소가 다른 개체에서 참조 중이라도 삭제가 수행된다.
- RESTRICT  : 삭제할 요소가 다른 객체에서 참조 중일 경우 삭제가 취소된다.

##### DCL 

- COMMIT : 명령어로 수행된 결과를 실제 물리적 디스크로 저장하고, 명령어로 수행을 성공적으로 완료하였음을 선언한다.
- ROLLBACK : 명령어로 수행에 실패하였음을 알리고, 수행된 결과를 원상복귀 시킨다.
- GRANT : 데이터베이스 사용자에게 사용 권한을 부여한다.
- REVOKE : 데이터 베이스 사용자로부터 사용 권한을 취소한다.

#### 하위 질의, SQL연산자

##### 개념   
- 하위 질의문은 하위 질의를 먼저 처리하고 검색된 결과는 상위 질의에 적용되어 검색된다.

##### 논리 연산자 설정  
- AND : 이면서, 그리고 조건
- OR : 이거나, 또는 조건
- NOT : 부정 조건

##### BETWEEN ~ AND  
- 구간 값 조건식이다.
- 점수 BETWEEN 90 AND 95은 90이상에서 95이하 까지의 범위를 의미한다.
- WHERE 점수 >= 90 AND 점수 <= 95로 표현할 수 있다.

#### 트랜잭션 

##### 특성 (ACID)

- 원자성(Atomicity) : 완전하게 수행완료되지 않으면 전혀 수행되지 않아야한다.
- 일관성(Consistency) : 시스템의 고정요소는 트랜잭션 수행 전후에 같아야 한다.
- 격리성(Isolation) : 트랜잭션 실행시 다른 트랜잭션의 간섭을 받지 않아야 한다.
- 영속성(Durability) : 트랜잭션의 완료 결과가 데이터 베이스에 영구히 기억된다.

##### 상태 

- 활동(Active) : 초기상태로 트랜잭션이 실행을 시작했거나 실행중인 상태
- 부분완료 (Partially Committed) : 트랜잭션의 마지막 연산이 실행된 직후의 상태로 모든 연산의 처리는 끝났지만 최종결과를 데이터베이스에 반영하지않은 상태
- 철회 (Aborted) :트랜잭션이 실행에 실패하여 Roolback연산을 수행한 상태
- 완료 (Committed) : 트랜잭션이 실행을 성공적으로 연산을 수행한 상태

#### 뷰 

- 뷰의 생성시 CREATE문, 검색시 SELECT문을 사용한다.
- 뷰의 정의 변경 시 ALTER문을 사용할 수 없고 DROP문을 이용한다.
- 뷰를 이요한 또 다른 뷰의 생성이 가능하다.
- 하나의 뷰 제거 시 그 뷰를 기초로 정의된 다른 뷰도 함께 삭제된다.
- 뷰에 대한 조작에서 삽입, 갱신, 삭제 연산은 제약이 따른다.
- 뷰가 정의된 기본 테이블에 제거되면 뷰도 자동적으로 제거 된다.

#### 인덱스

- 데이터베이스 성능에 많은 영향을 주는 DBMS의 구성요소로 테이블과 클러스터에 연관되어 독립적인 저장 공간을 보유하며, 데이터베이스에 저장된 자료를 더우 빠르게 조회하기 위하여 별도로 구성한 순서 데이터를 말한다.
- 데이터베이스 대부분에서 테이블을 삭제하면 인덱스도 같이 삭제된다.
